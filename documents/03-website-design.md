# 03 — 網站設計規格

## 設計原則

1. **數據優先** — 所有結論都由數據和圖表支持
2. **互動探索** — 使用者可調整參數即時看到結果變化
3. **漸進揭露** — 從簡單結論到詳細數據，層層深入
4. **手機友善** — 響應式設計，圖表在小螢幕上也能清楚閱讀
5. **風格一致** — 延續現有 index.html 的視覺語言

## 視覺設計

### 字型
- 系統字型堆疊：`-apple-system, "Microsoft JhengHei", sans-serif`
- 數字使用等寬字型確保對齊

## 頁面規格

### 首頁 (/)
**目的**：說明專案動機，引導使用者進入各個分析工具

**內容區塊**：
1. Hero 區 — 標題、一句話描述、CTA 按鈕
2. 核心問題 — 用大字體呈現「借 150 萬買 ETF，7 年後會怎樣？」
3. 四個標的卡片 — 各標的名稱、性質、一句話結論、連結到詳細頁面
4. 快速結果預覽 — 以 0050 為例的關鍵數據
5. 使用指南 — 如何使用本站的分析工具
6. 免責聲明

### 試算工具頁 (/calculator/{ticker}/)
**目的**：互動式借貸投資回測試算

**參數面板**（可調整）：
- 貸款金額：滑桿 + 數字輸入（50萬~300萬，預設150萬）
- 年利率：滑桿 + 數字輸入（1%~8%，預設4.23%）
- 還款期數：下拉選單（36/48/60/72/84期，預設84期）
- 買入日期：日期選擇器（限資料可用範圍）
- 股利策略：單選（再投入/用於還款/混合）
- 未來推估：單選（歷史報酬率/自訂報酬率/不推估）

**結果展示**：
1. 結論卡片 — 大字體顯示最終損益
2. 摘要指標卡片組 — 總利息、總交易成本、總股利、最終市值等
3. 淨資產走勢圖 — 折線圖，X軸為月份，Y軸為淨資產
4. 持股市值 vs 剩餘貸款圖 — 雙線圖
5. 逐月明細表 — 延續現有表格設計
6. 股利事件明細表

### 多標的比較頁 (/calculator/compare/)
**目的**：同一組貸款條件下，比較四種標的的表現

**展示方式**：
1. 共用參數面板
2. 四條淨資產走勢線疊在同一張圖上
3. 並排比較卡片（最終損益、風險指標等）
4. 比較表格

### 歷史期間分析頁 (/analysis/historical/)
**目的**：展示不同進場時間的回測結果

**展示方式**：
1. 熱力圖 — X軸為進場年月，Y軸為還款期數，顏色為淨資產
2. 勝率/敗率統計
3. 最佳/最差進場時間點案例分析
4. 滑桿選擇特定進場時間，展示完整模擬結果

### 情境模擬頁 (/analysis/scenarios/)
**目的**：展示不同市場假設下的預期結果

**展示方式**：
1. 五種情境的結果比較
2. 扇形圖（Fan Chart）— 展示可能的結果分布範圍
3. 各情境的關鍵風險指標

### 知識指南頁 (/guide/)
**目的**：提供借貸投資的基礎知識

**內容**：純靜態文字 + 圖表，無互動元素

```
│   └── compare/index.html
├── analysis/

│       ├── data-loader.js      ← 資料載入模組
│       └── ui-components.js    ← 共用 UI 元件

│   ├── 0050_daily_prices.json
│   ├── 0056_daily_prices.json
│   ├── 00631L_daily_prices.json
│   ├── 00679B_daily_prices.json
│   └── dividends/
│       ├── 0050_dividends.json
│       └── ...
└── documents/                  ← 規劃文件（不部署）
```

### 第三方函式庫
- **Chart.js** (CDN) — 互動式圖表
- 無其他外部依賴，保持輕量

### 效能考量
- 價格資料 JSON 檔案使用 lazy loading
- 大量計算使用 Web Worker 避免阻塞 UI
- 圖表使用 Canvas 而非 SVG 以提升大資料量效能
