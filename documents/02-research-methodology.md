# 02 — 研究方法論

## 基本模型：借貸投資模擬

### 模擬流程

```
Day 0: 貸款撥款 → 全額買入標的 ETF
       ↓
Month 1~N: 每月循環
  ├─ 檢查是否有股利入帳 → 依策略處理（再投入/用於還款）
  ├─ 檢查是否有公司行動（分割/合併）
  ├─ 計算本月應還款金額（本金 + 利息）
  ├─ 若現金不足 → 賣出最少需要的股數
  ├─ 支付月還款
  └─ 記錄：持股數、市值、淨資產、現金餘額
       ↓
Month N (最後一期): 還清貸款
  ├─ 計算剩餘股票市值 + 剩餘現金
  ├─ 扣除所有成本（利息、交易費、稅）
  └─ 得出最終淨損益
```

### 關鍵假設

| 假設項目 | 設定值 | 說明 |
|----------|--------|------|
| 買入時機 | 撥款日收盤價 | 假設當日全額買入 |
| 賣出時機 | 每月 13 日（或最近交易日） | 在還款日前賣出 |
| 成交價格 | 收盤價 | 簡化假設，實務可能有價差 |
| 買入手續費 | 0.1425% | 法定上限，實務可能有折扣 |
| 賣出手續費 | 0.1425% | 同上 |
| 證交稅 | 0.1% | ETF 優惠稅率 |
| 股利處理 | 再投入買股 | 除非接近還款日 |
| 交易單位 | 依時期而定 | 2005-03-01 後可零股交易；2005-03-01 前僅能整張（1,000 股）交易，詳見下方說明 |

### 交易單位的時期差異

回測模擬需依據歷史上零股交易的開放時間，採用不同的賣出邏輯：

| 時期 | 交易制度 | 回測邏輯 |
|------|---------|---------|
| **2005-03-01 之前** | 僅整張交易（1,000 股） | 每月賣出 `Math.ceil(需求金額 / 每張淨收入)` 張，多餘現金留存至下月一併計算（不買回，因零頭不足一張） |
| **2005-03-01 之後** | 盤後零股交易開放 | 每月賣出 `Math.ceil(需求金額 / 每股淨收入)` 股，幾乎無閒置現金 |

> **說明**：台灣盤後零股交易自 2005-03-01 起開放，2020-10-26 再開放盤中零股交易。由於本模型使用收盤價計算且需考量 T+2 交割（提前 2~3 天賣出），盤中與盤後零股在模型中無差異。
>
> **影響範圍**：僅 0050 在 2003-07 至 2005-02 期間的回測受此限制，需以整張交易模擬。此時 0050 股價約 37~58 元，一張約 37,000~58,000 元，月還款約 20,000 元，每月賣出一張會產生約 17,000~38,000 元的閒置現金，對績效有一定拖累。

## 分析維度一：歷史期間回測

### 滾動回測法
- 以每個月的第一個交易日作為假想的貸款撥款日
- 模擬完整的 84 期（7年）還款週期
- 記錄每個進場時間點的最終結果

### 分析範例（以 0050 為例）
```
進場 2010-01: 經歷歐債危機 → 結果 A
進場 2015-01: 經歷中美貿易戰 → 結果 B
進場 2018-01: 經歷 COVID-19 → 結果 C
進場 2020-03: 疫情低點進場 → 結果 D
...
```

### 產出指標
- 勝率：多少比例的進場時間點最終獲利
- 平均報酬率與中位數報酬率
- 最佳/最差進場時間點及其結果
- 還款期間最大淨資產回撤

## 分析維度二：未來推估 — Bootstrap 歷史模擬法

### 方法論背景

本平台採用 **Non-parametric Bootstrap Monte Carlo Simulation**（無母數自助抽樣蒙地卡羅模擬）作為核心的未來推估方法。此方法由統計學家 Bradley Efron（Stanford, 1979）提出，後被廣泛應用於金融風險管理（Basel III 的歷史模擬法）和退休金規劃（FIRECalc、William Bengen 的 4% Rule 研究）。

### 為什麼不用傳統方法

| 方法 | 問題 |
|------|------|
| 固定成長率推估（mean） | 每月都漲同樣幅度，不切實際；近期牛市會高估，盤整期會低估 |
| 常態分佈假設（mean ± σ） | 假設報酬率為常態分佈，低估了極端事件（厚尾風險）；三條線仍然是平滑曲線 |
| 預設五種情境（牛市/熊市/盤整...） | 參數是人為設定的，主觀性強；無法量化每種情境的發生機率 |

### Bootstrap 模擬法的核心理念

**「不預測未來，但假設未來的月漲跌幅會落在歷史觀測的範圍內」**

具體而言：
1. 從標的的完整歷史中收集所有月漲跌幅（以百分比計算，避免因股價水位不同而產生偏差）
2. 這些歷史月漲跌幅構成一個「亂數表」（經驗分佈）
3. 模擬未來時，每個月的漲跌幅從亂數表中隨機抽取（有放回抽樣）
4. 重複抽樣 5,000 次，產生 5,000 條不同的未來走勢路徑
5. 從 5,000 條路徑中，以百分位數定義三種情境

### 模擬流程

```
Step 1: 收集歷史月報酬率
  ├─ 將完整歷史價格按月分組，取每月最後交易日收盤價
  ├─ 計算相鄰月份報酬率: r[m] = price[m+1] / price[m] - 1
  ├─ 處理特殊事件：若某月跨越分割日，報酬率需乘以分割比例修正
  └─ 產出：歷史月報酬率陣列（如 0050 約 270 個月、00679B 約 108 個月）

Step 2: Bootstrap 抽樣（5,000 次）
  ├─ 確定需推估的期數 N（= 貸款總期數 - 已有實際資料的期數）
  ├─ 每次模擬：從歷史月報酬率中隨機抽取 N 個值（有放回）
  ├─ 用抽取的報酬率序列推算未來每月的股價
  └─ 對每條路徑執行完整的 BacktestEngine.simulate()

Step 3: 統計百分位數
  ├─ 對 5,000 次模擬的最終淨資產排序
  ├─ 樂觀情境 = P75（75% 的模擬結果比這個差）
  ├─ 基準情境 = P50（中位數）
  ├─ 悲觀情境 = P25（25% 的模擬結果比這個差）
  └─ 進階圖表額外顯示 P10 和 P90
```

### 方法優勢

1. **無分佈假設**：不假設報酬率為常態分佈，自然保留了真實的偏態和厚尾
2. **有漲有跌**：每條模擬路徑都有真實的漲跌起伏，不是平滑曲線
3. **保留極端值**：歷史上的暴漲暴跌都有機率被抽到，風險評估更真實
4. **客觀無主觀**：不需要人為設定牛市/熊市的報酬率和波動率
5. **捕捉時序風險**：不同的漲跌順序會產生不同結果，這正是借貸投資的核心風險（Sequence of Returns Risk）

### 技術參數

| 參數 | 設定值 | 說明 |
|------|--------|------|
| 模擬次數 | **5,000 次** | 足以穩定估計 P10~P90（Davison & Hinkley, 1997 建議 1,000~10,000） |
| 抽樣方式 | 有放回隨機抽樣（Bootstrap） | Efron (1979) 標準方法 |
| 報酬率計算 | 月漲跌百分比 | 避免因股價水位不同而產生偏差 |
| 分割修正 | 分割月報酬率 × 分割比例 | 防止 0050 在 2025-06 出現 -75% 假報酬 |
| 執行環境 | Web Worker | 不阻塞 UI，約 2~3 秒完成 |

### 三情境定義

| 情境 | 百分位數 | 意義 |
|------|---------|------|
| **樂觀** | P75 | 75% 的模擬結果比這個差，25% 比這個好 |
| **基準** | P50 | 中位數，最可能的結果 |
| **悲觀** | P25 | 25% 的模擬結果比這個差 |

進階信心區間圖表額外顯示：
- **P90**：只有 10% 的機率比這個好（極度樂觀邊界）
- **P10**：只有 10% 的機率比這個差（極度悲觀邊界）

### 樣本數差異

各標的可用的歷史月報酬率數量不同，影響模擬的代表性：

| 標的 | 掛牌日 | 約可用月數 | 涵蓋的市場週期 |
|------|--------|-----------|---------------|
| 0050 | 2003-06 | ~270 個月 | 2008 金融海嘯、歐債危機、COVID-19、2022 升息 |
| 0056 | 2007-12 | ~220 個月 | 2008 金融海嘯（部分）、歐債、COVID-19、升息 |
| 00631L | 2014-10 | ~136 個月 | 2015 股災、COVID-19、2022 升息 |
| 00679B | 2017-01 | ~108 個月 | COVID-19、2022 升息（債券大跌） |

> **注意**：00679B 和 00631L 的歷史月數較少，模擬結果的代表性相對不足。UI 上會顯示使用的歷史月數，讓使用者自行判斷。

### 學術參考

- Efron, B. (1979). *Bootstrap methods: Another look at the jackknife*. Annals of Statistics.
- Davison, A. C. & Hinkley, D. V. (1997). *Bootstrap Methods and their Application*. Cambridge University Press.
- Basel Committee on Banking Supervision — Historical Simulation approach for VaR calculation.
- Bengen, W. P. (1994). *Determining Withdrawal Rates Using Historical Data*. Journal of Financial Planning.

### 方法限制

1. **獨立同分佈假設**：Bootstrap 假設每月報酬率是獨立的，忽略了動量效應和均值回歸
2. **歷史非未來**：若未來出現前所未見的市場結構（如 AI 泡沫），歷史分佈可能不適用
3. **簡化股利模型**：Bootstrap 主路徑模擬中不含股利再投入（速度優先），三情境主模擬則包含完整股利處理
4. **無跨資產相關性**：各標的獨立模擬，未考慮資產間的動態相關性

## 分析維度三：還款壓力測試

### 現金流斷裂風險
- 定義：某月賣出所有剩餘持股仍無法支付月還款
- 衡量指標：在各情境下，現金流斷裂的機率
- 觸發條件：股價跌幅 × 已賣出比例 → 剩餘股數不足

### 心理壓力指標
- 帳面虧損超過貸款金額 50% 的月份數
- 連續帳面虧損的最長月份數
- 需要額外掏錢還款（自掏腰包）的月份數

### 安全邊際計算
- 在各情境下，保留多少現金儲備可避免斷裂
- 每月額外投入多少金額可顯著降低風險
