## Phase 0：資料準備

### 目標
使用 FinMind API 收集四個標的的完整歷史價格與股利資料

### 工作項目
- [ ] 建立資料收集腳本 `scripts/fetch-data.js`（Node.js）
  - 使用 FinMind API（免費方案，600 req/hr，已驗證可取得掛牌日起完整資料）
  - Token 透過環境變數 `FINMIND_TOKEN` 傳入
- [ ] 收集四個標的歷史日價格（`TaiwanStockPrice`，共 4 次 API 請求）
  - 0050：2003-06-30 起（~5,589 筆）
  - 0056：2007-12-26 起（~4,400 筆）
  - 00631L：2014-10-31 起（~2,700 筆）
  - 00679B：2017-01-17 起（~2,200 筆，注意此標的在 TPEx 掛牌）
- [ ] 收集股利紀錄（共 3 次 API 請求）
  - 0050, 0056：使用 `TaiwanStockDividend` dataset
  - 00679B：使用 **`TaiwanStockDividendResult`** dataset（債券 ETF 需用不同 dataset）
  - 00631L：無配息（已確認）
- [ ] 轉換為專案 JSON 格式並存入 `data/`
- [ ] 建立 `data/corporate_actions/0050_actions.json`（2025-06-18 一拆四）
- [ ] 交叉驗證：比對現有 index.html 中的 0050 價格（2024-04 ~ 2026-03）
- [ ] 建立 `data/metadata.json` 記錄更新時間

### 產出
- `data/` 目錄下四個標的的日價格 JSON 檔
- `data/dividends/` 目錄下三個標的的股利 JSON（00631L 無配息）
- `data/corporate_actions/` 公司行動紀錄
- `scripts/fetch-data.js` 可重複執行的資料收集腳本

## Phase 4：情境模擬（Bootstrap 歷史模擬法）

### 目標
以 Bootstrap Monte Carlo 模擬提供客觀的未來推估與風險量化

### 方法論
採用 Non-parametric Bootstrap（Efron, 1979）：從各標的完整歷史月報酬率中有放回隨機抽樣，模擬 5,000 次完整還款週期，以百分位數定義三情境（P25 悲觀 / P50 基準 / P75 樂觀）。

### 工作項目
- [x] 實作 Bootstrap 模擬引擎（`scenario-engine.js` + `monte-carlo-worker.js`）
- [x] 各計算器頁面整合三情境切換（樂觀/基準/悲觀）
- [x] 實作蒙地卡羅信心區間圖表（P10~P90 扇形帶）
- [ ] 建立情境模擬頁面 (`/analysis/scenarios/`)
  - Bootstrap 結果的視覺化展示
  - 扇形圖（Fan Chart）展示 P10~P90 信心區間
  - VaR / CVaR 計算與展示
- [ ] 建立風險量化分析頁面 (`/analysis/risk/`)
  - 最大回撤分析
  - 夏普比率比較
  - 風險調整後報酬比較

### 產出
- Bootstrap 三情境模擬引擎（已完成）
- 蒙地卡羅信心區間圖表（已完成）
- 情境模擬頁面（待實作）
- 風險分析頁面（待實作）
